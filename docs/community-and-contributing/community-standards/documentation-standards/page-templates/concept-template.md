# Concept Template

{% hint style="info" %}
**Concept Template**

Use for: feature overviews, explanations of platform constructs, background on how something works. A concept page should never contain steps — those belong in a task page. The concept page is typically the parent container in GitBook, with task pages as children beneath it.

**Concept Page Title:** _Name the thing, not the action — e.g. "Policy Integrity Tests", "TrustChain", "Verifiable Credentials"_
{% endhint %}

One to two sentences. Plain language definition of what this is.

#### The problem it solves

What would happen without this feature or capability. Why it exists.

#### How it works

The mental model. How to think about this conceptually. Not steps — those belong in the task page.

#### Key distinctions

How this differs from something it might be confused with. Remove this section if there is nothing to distinguish.

#### Related

* Task: how to use this
* Reference: full specification
* Concept: related concept
